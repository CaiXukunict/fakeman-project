【1】输入方式：主动 → 被动

    旧版流程：
    ┌────────┐   写入   ┌──────────┐   读取   ┌────────┐
    │ 用户    │─────────>│  JSON文件 │<─────────│ Ability │
    └────────┘          └──────────┘          └────────┘
                                                   ↓
                                              ┌────────┐
                                              │ 系统    │
                                              └────────┘

    新版流程：
    ┌────────┐   直接传入   ┌────────┐
    │ 用户    │─────────────>│ 系统    │
    └────────┘              └────────┘

    优势：
    ✓ 减少2个中间步骤
    ✓ 零文件I/O延迟
    ✓ 更可靠的数据传递
    ✓ 简化的架构

【2】仪表盘系统

    新增完整的实时仪表盘：

【3】信息完整度提升

    旧版显示：
    - 目的: 只有描述和bias
    - 手段: 只有描述和重要性
    
    新版显示：
    - 目的: 描述 + bias + 可达成性 + 预期满足 + 类型 + 状态
    - 手段: 描述 + 重要性 + 成功率 + 执行统计 + 目标目的

【4】命令系统改进

    旧版：
    - 输入 "status" 或 "s"
    - 输入 "purposes" 或 "p"
    - 混合在对话中，可能误触发
    
    新版：
    - 输入 "/dashboard" 或 "/d"
    - 输入 "/purposes" 或 "/p"
    - 使用 / 前缀区分命令和对话
    - 不会误触发命令

  或
  启动交互模式.bat

适用场景：
  - 后台持续运行
